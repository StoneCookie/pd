Задача:

Создание портала для сотрудников сервиса Яндекс.Деньги, с помощью которого можно было бы нематериально мотивировать их к работе в формате игры. Цели для сотрудников ставятся в формате квестов различной длительности и сложности. За выполнение квестов сотрудникам будут начисляться игровые баллы\валюта. Должны быть реализованы личные кабинеты для администратора, руководителя, эксперта, сотрудника и гейм мастера.

Цели, которые нами движат:

  • Повысить вовлеченность сотрудников
  • Моральный дух сотрудников
  • Разнообразить рабочий процесс

Для выполнения задачи необходимо разработать инструмент, в котором будут реализованы следующие модули:

Конструктор заданий

Конструктор должен содержать следующие поля:

  • Выбор подразделения, для которого ставится задача
  • Таймер длительности задания (дедлайн)-в случае если выбран тип ежедневное задание
  • Выбор сложности задания, 3 уровня сложности
  • Поле для описания задачи
  • Возможность добавлять тэги к задачам
  • Список подзадач задания
  • Награда (зависит от сложности, времени и количества подзадач, а также типа задания)
  • Кнопка закрыть и добавить задачу
  • Поле для написание заголовка для задачи
  • Выбор дней, в которые данная задача должна действовать
  • Индивидуальная/Командная задача выбор
  • Тип задания:
    1. Квест
    2. Ежедневная задача
  • В случае если выбран квест, то значение левела, с которого открывается данный квест

Конструктор вопросов и задач

Конструктор должен содержать следующие поля:

  • Название вопроса или задачи
  • Описание вопроса(условие)
  • Варианты ответа(один из)
    1. Один выбор
    2. мультивыбор
    3. Ввод числа
    4. Ввод текста
  • Уровни сложности, 5 видов
  • Поле для загрузки картинки (может быть пустым)
  • Выбор тематики вопроса
  • Добавление тэга к вопросу

Конструкто ачивок

Конструктор должен содержать следующие поля:

  • Название ачивки
  • Описание ачивки
  • Описания условий получения ачивки
  • Вставки изображения ачивки
  • Описания наград за получение ачивки

Конструкто соревнований

Конструктор должен содержать следующие поля:

  • Участников соревнования
  • Критерий, по которому проходит соревнование
  • Поле с выбором соперника, должна быть возможность поиска сотрудников по имени и фамилии
  • Длительность соревнования
  • Награда

Конструкто товаров для магазина

Конструктор должен содержать следующие поля:

  • Название товара
  • Выбор категории товара
  • Фото товара
  • Условия получения товара
  • Описание товара
  • Количество товара
  • В зависимости от категории товара

Конструкто тестов

Конструктор должен содержать следующие поля:

Тест состоит из блоков, каждый создается отдельно

  • Тематика вопросов
  • Число вопросов
  • Коэффициент блока, при расчете итоговой оценки за текс
  • Поле с выбором вопросов
  • Перенести тематику и количество вопросов из предыдущего блока
  • Настройка –можно покидать страницу с тестом или нет
  • астройка по возможности выбирать один или несколько вариантов ответа
  • Есть возможность перешагивать от одного вопроса к другому и возвращаться к вопросу, на который не смог ответить

Конструкто создания команд

Конструктор должен содержать следующие поля:

  • Название команды
  • Максимальное количество участников
  • Наименование подразделения, из которого выбираются участники
  • Поле со списком всех работников по подразделению, должна быть возможность поиска сотрудников по имени и фамилии
  • Описание товара
  • Количество товара
  • В зависимости от категории товара

Конструктор создания тестовых заданий для собеседований

Конструктор должен содержать следующие поля:

  • Тест состоит из списка вопросов. В зависимости от требований Ответы можно:
    1. выбрать из списка предложенных
    2. составить ответ руками
  • Есть возможность задать настройки на просмотр:
    1. сколько времени сотрудник потратил на выполнение заданий
    2. закрывал или нет окно с тестом
